Day one of learning/teaching Web Development

When I began my Comp Sci degree, I was initially hoping to evade the realm of Web Design. I wasn't that keen on making things aestitically pleasing, but instead wanted to focus on making stuff work. Alas, with the rise in popularity of smartphones, interacting with the largest population necessitates at least some proficiency with web development. The web-dev courses I took at UPeople demonstrated to me that web development has changed significantly from my first foray into it nearly 20 years ago. It's not as clunky, limited, fragile, and totally annoying as it was; actually it's pretty fun. This blog will serve two purposes: As a documentation process of my web development... uhh... development; and secondly, as an introduction to web-development for anyone reading.

First of all, the concepts behind web development are incredibly simple. Anyone can do it, no problem. You are definitely capable of it, and if you haven't done any programming at all, this is a great place to start! You might even like it... The architecture of a web page today consists of three main parts:

  • an HTML file, which describes the content of a web page
  • a CSS stylesheet, that describes how the content should appear in your browser
  • and some JavaScript or Jquery, either embedded in the HTML file or in it's own script.js file

The main point to take away from this architecture is the separation of content and display. The HTML {Hyper Text Markup Language} file describes the structure and content of a web page; it says "this content is of this type, and here's how it all relates". This may sound abstract at first, and it is, but it's also super approachable. The type is described by tags that enclose the content:
